# Skills团队共享

## 定义

Skills 团队共享是指把个人或小组沉淀的 Skills 发布到团队公共空间，使其他成员能够浏览、安装、使用并跟随更新。它把原本分散在个人环境中的 prompt、流程、脚本、工具说明和操作经验，转化为团队可复用的 [[llm-agent/concepts/团队知识资产|团队知识资产]]。

## 为什么重要

在 LLM/Agent 工作中，很多效率来自“怎么问、怎么拆任务、怎么调用工具、怎么稳定复现流程”。如果这些能力只停留在个人上下文中，团队会反复造轮子。通过 [[llm-agent/concepts/Skills Hub|Skills Hub]] 共享后，团队可以把成熟经验沉淀为统一入口，并让成员在同一套能力基础上协作。

更凝练的结论见：[[llm-agent/notes/Skills团队共享重点|Skills团队共享重点]]。

## 典型能力

- 上传或发布个人 Skill
- 浏览团队内已有 Skills
- 安装他人共享的 Skills
- 通过 [[llm-agent/concepts/Skill版本同步|Skill版本同步]] 获取更新
- 通过成员与权限管理控制使用和维护范围

## 设计要点

- 可发现：成员需要能快速搜索到相关 Skill。
- 可理解：Skill 应说明适用场景、输入要求、输出结果和边界。
- 可维护：Skill 应有版本、维护人、更新记录和失效处理机制。
- 可治理：涉及内部资料、账号权限和密钥时，需要清晰的安全边界。
