# Skills团队共享重点

## 核心判断

[[llm-agent/concepts/Skills团队共享|Skills团队共享]] 最重要的是：把个人经验变成团队可复用、可维护、可治理的 [[llm-agent/concepts/团队知识资产|团队知识资产]]。

它的重点不只是分享一个 Skill 文件，而是让团队成员沉淀的 prompt、流程、脚本、工具说明和操作经验进入公共空间，能被别人发现、安装、使用，并且后续还能通过 [[llm-agent/concepts/Skill版本同步|Skill版本同步]] 持续维护。

## 四个关键点

- 可复用：避免每个人重复造轮子。
- 可发现：通过 [[llm-agent/concepts/Skills Hub|Skills Hub]] 让成员能找到已有能力。
- 可维护：有版本、维护人、更新记录，避免 Skill 过期或漂移。
- 可治理：权限、数据、安全边界清楚，避免共享带来风险。

## 一句话总结

Skills 团队共享的核心价值，是让个人效率变成团队能力。
